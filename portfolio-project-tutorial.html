Lesson 1: Project Page Structure & Semantic HTML

Understanding Project Page Architecture

When creating individual project pages, you need to think about the user's journey and information hierarchy. A project page serves multiple purposes:

Project Page Goals:
  • Showcase Work: Display your project visually with high-quality images/videos
  • Explain Process: Detail your technical approach and problem-solving
  • Demonstrate Skills: Show code, tools, and methodologies used
  • Provide Context: Explain the project's purpose and constraints
  • Enable Navigation: Clear paths back to portfolio and between projects

HTML5 Semantic Structure

Modern HTML5 provides semantic elements that describe content meaning, not just appearance. This helps with SEO, accessibility, and code maintainability:

<!-- Semantic elements describe content purpose -->
<header>    <!-- Page header with navigation -->
<main>      <!-- Primary content area -->
  <article> <!-- Self-contained content (the project) -->
    <header> <!-- Project title and meta info -->
    <section><!-- Project overview -->
    <section><!-- Technical details -->
    <section><!-- Media gallery -->
  </article>
</main>
<aside>     <!-- Related projects or sidebar info -->
<footer>    <!-- Page footer -->

Information Architecture Principles

Organizing Project Information:
  • F-Pattern Reading: Users scan left-to-right, then down - place key info accordingly
  • Progressive Disclosure: Start with overview, then dive into details
  • Consistent Navigation: Clear breadcrumbs and "back to portfolio" links
  • Mobile-First: Design for small screens first, then enhance for desktop

Lesson 2: Interactive Media Gallery Systems

Understanding Media Gallery Patterns

Media galleries are essential for showcasing visual work, but they present unique UX challenges. Users expect intuitive navigation, fast loading, and accessibility across devices.

Gallery Design Principles:
  • Thumbnail Overview: Show all media at a glance for quick selection
  • Full-Size Viewing: Provide immersive viewing without page navigation
  • Progressive Loading: Load thumbnails first, full-size on demand
  • Keyboard Navigation: Arrow keys, escape, and tab support
  • Mobile-Friendly: Touch gestures and responsive sizing

CSS Grid for Responsive Layouts

CSS Grid provides powerful tools for creating responsive gallery layouts that adapt to different screen sizes:

/* Auto-fit creates responsive columns */
.gallery-grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(150px, 1fr));
    gap: 15px;
}

/* Auto-fill vs Auto-fit */
grid-template-columns: repeat(auto-fill, minmax(150px, 1fr));
/* ^ Creates empty columns to fill space */

grid-template-columns: repeat(auto-fit, minmax(150px, 1fr));
/* ^ Expands existing columns to fill space */

/* Responsive breakpoints */
@media (max-width: 768px) {
    .gallery-grid {
        grid-template-columns: repeat(2, 1fr);
    }
}

Modal Design Patterns

Modals provide full-screen viewing without navigating away from the page. Key considerations:

Modal Best Practices:
  • Overlay Background: Semi-transparent backdrop focuses attention
  • Escape Routes: Multiple ways to close (X button, escape key, click outside)
  • Focus Management: Trap focus within modal for accessibility
  • Body Scroll Lock: Prevent background scrolling when modal is open

JavaScript Event Handling

JavaScript Gallery Functionality

class ProjectGallery {
    constructor() {
        this.currentIndex = 0;
        this.mediaItems = [];
        this.init();
    }

    init() {
        this.setupGallery();
        this.setupModal();
        this.setupKeyboardNavigation();
    }

    setupGallery() {
        const thumbnails = document.querySelectorAll('.gallery-thumb');
        this.mediaItems = Array.from(thumbnails).map(thumb => ({
            src: thumb.dataset.fullSrc,
            alt: thumb.alt,
            type: thumb.dataset.type || 'image'
        }));

        thumbnails.forEach((thumb, index) => {
            thumb.addEventListener('click', () => this.openModal(index));
        });
    }

    openModal(index) {
        this.currentIndex = index;
        const modal = document.querySelector('.gallery-modal');
        const content = modal.querySelector('.modal-content');
        
        this.loadMedia(content, this.mediaItems[index]);
        modal.style.display = 'block';
        document.body.style.overflow = 'hidden';
    }

    loadMedia(container, item) {
        if (item.type === 'video') {
            container.innerHTML = `<video controls>
                <source src="${item.src}" type="video/mp4">
            </video>`;
        } else {
            container.innerHTML = `<img src="${item.src}" alt="${item.alt}">`;
        }
    }
}

Copy-to-Clipboard Functionality

Make it easy for other developers to use your code:

// JavaScript for copy functionality
function copyToClipboard(text) {
    navigator.clipboard.writeText(text).then(() => {
        showToast('Code copied to clipboard!');
    }).catch(err => {
        // Fallback for older browsers
        const textArea = document.createElement('textarea');
        textArea.value = text;
        document.body.appendChild(textArea);
        textArea.select();
        document.execCommand('copy');
        document.body.removeChild(textArea);
        showToast('Code copied to clipboard!');
    });
}

Lesson 4: Responsive Design & Mobile-First Development

Understanding Responsive Design Philosophy

Responsive design isn't just about making things smaller‚Äîit's about creating optimal experiences across all devices. With mobile traffic often exceeding desktop, mobile-first design is essential.

Responsive Design Principles:
  • Mobile-First: Design for small screens first, then enhance for larger screens
  • Flexible Grids: Use relative units (%, fr, em) instead of fixed pixels
  • Flexible Images: Images that scale with container size
  • Media Queries: Apply different styles based on screen characteristics
  • Touch-Friendly: Larger touch targets and gesture support

Lesson 5: Client-Side Security & Password Protection

Understanding Static Site Security Limitations

Static sites like GitHub Pages don't have servers to handle authentication, but you can implement client-side protection for basic privacy needs. This isn't military-grade security, but it's sufficient for protecting portfolio work from casual browsing.

Client-Side Security Concepts:
  • Obfuscation: Making content difficult to access casually
  • Password Hashing: Never store plain text passwords
  • Session Management: Remember authentication during browsing
  • User Experience: Make legitimate access smooth and clear
  • Graceful Degradation: Handle failed attempts professionally

Cryptographic Hashing with Web Crypto API

Modern browsers provide secure hashing functions through the Web Crypto API:

// Generate password hash (do this once, store the result)
async function generatePasswordHash(password) {
    const encoder = new TextEncoder();
    const data = encoder.encode(password);
    const hashBuffer = await crypto.subtle.digest('SHA-256', data);
    const hashArray = Array.from(new Uint8Array(hashBuffer));
    return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
}

// Example usage:
// const hash = await generatePasswordHash('myClientPassword123');
// console.log(hash); // Store this hash in your code, not the password

// Verify password against stored hash
async function verifyPassword(inputPassword, storedHash) {
    const inputHash = await generatePasswordHash(inputPassword);
    return inputHash === storedHash;
}

Session Management

Use browser storage to remember authentication during the session:

// Session storage (cleared when browser tab closes)
class SessionAuth {
    constructor(pageId) {
        this.pageId = pageId;
        this.storageKey = `auth_${pageId}`;
    }
    
    setAuthenticated() {
        sessionStorage.setItem(this.storageKey, 'true');
        sessionStorage.setItem(`${this.storageKey}_time`, Date.now().toString());
    }
    
    isAuthenticated() {
        const auth = sessionStorage.getItem(this.storageKey);
        const authTime = sessionStorage.getItem(`${this.storageKey}_time`);
        
        // Optional: Add expiration (e.g., 1 hour)
        const oneHour = 60 * 60 * 1000;
        if (authTime && Date.now() - parseInt(authTime) > oneHour) {
            this.clearAuth();
            return false;
        }
        
        return auth === 'true';
    }
    
    clearAuth() {
        sessionStorage.removeItem(this.storageKey);
        sessionStorage.removeItem(`${this.storageKey}_time`);
    }
}

Error Handling & Rate Limiting

Implement graceful error handling and basic rate limiting:

class PasswordProtection {
    constructor() {
        this.maxAttempts = 5;
        this.attempts = 0;
        this.lockoutTime = 15 * 60 * 1000; // 15 minutes
        this.lastAttempt = 0;
    }
    
    async handlePasswordSubmit(password) {
        // Check if locked out
        if (this.isLockedOut()) {
            this.showError('Too many failed attempts. Please wait 15 minutes.');
            return false;
        }
        
        // Verify password
        if (await this.verifyPassword(password)) {
            this.resetAttempts();
            this.showSuccess();
            return true;
        } else {
            this.attempts++;
            this.lastAttempt = Date.now();
            this.showError(`Incorrect password. ${this.maxAttempts - this.attempts} attempts remaining.`);
            return false;
        }
    }
    
    isLockedOut() {
        if (this.attempts >= this.maxAttempts) {
            const timeSinceLastAttempt = Date.now() - this.lastAttempt;
            return timeSinceLastAttempt < this.lockoutTime;
        }
        return false;
    }
    
    showError(message) {
        const errorDiv = document.getElementById('password-error');
        errorDiv.textContent = message;
        errorDiv.style.display = 'block';
    }
}

// Password protection system
class PasswordProtection {
    constructor(pageId, correctPasswordHash) {
        this.pageId = pageId;
        this.correctPasswordHash = correctPasswordHash;
        this.isAuthenticated = false;
        this.init();
    }

    init() {
        // Check if already authenticated in this session
        const sessionAuth = sessionStorage.getItem(`auth_${this.pageId}`);
        if (sessionAuth === 'true') {
            this.showContent();
        } else {
            this.showPasswordPrompt();
        }
    }

    async hashPassword(password) {
        const encoder = new TextEncoder();
        const data = encoder.encode(password);
        const hashBuffer = await crypto.subtle.digest('SHA-256', data);
        const hashArray = Array.from(new Uint8Array(hashBuffer));
        return hashArray.map(b => b.toString(16).padStart(2, '0')).join('');
    }

    async checkPassword(inputPassword) {
        const hashedInput = await this.hashPassword(inputPassword);
        return hashedInput === this.correctPasswordHash;
    }

    showPasswordPrompt() {
        const protectedContent = document.querySelector('.protected-content');
        protectedContent.innerHTML = `
            <div class="password-overlay">
                <div class="password-prompt">
                    <h2>üîí Protected Content</h2>
                    <p>This project contains confidential work. Please enter the password to view.</p>
                    <div class="password-form">
                        <input type="password" id="password-input" placeholder="Enter password">
                        <button onclick="passwordProtection.submitPassword()">Access</button>
                    </div>
                    <div class="password-error" id="password-error"></div>
                </div>
            </div>
        `;

        // Add enter key support
        document.getElementById('password-input').addEventListener('keypress', (e) => {
            if (e.key === 'Enter') {
                this.submitPassword();
            }
        });
    }

    async submitPassword() {
        const input = document.getElementById('password-input');
        const errorDiv = document.getElementById('password-error');
        
        if (await this.checkPassword(input.value)) {
            sessionStorage.setItem(`auth_${this.pageId}`, 'true');
            this.showContent();
        } else {
            errorDiv.innerHTML = '‚ùå Incorrect password. Please try again.';
            input.value = '';
            input.focus();
        }
    }

    showContent() {
        const protectedContent = document.querySelector('.protected-content');
        protectedContent.innerHTML = `
            <div class="project-content">
                <!-- Your actual project content here -->
                <h2>Confidential Project Details</h2>
                <p>This is the protected content that requires authentication.</p>
                <div class="content-warning">
                    ‚ö†Ô∏è This content is confidential. Please do not share or redistribute.
                </div>
            </div>
        `;
    }
}
